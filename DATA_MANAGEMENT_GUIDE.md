# 用户档案与文件管理逻辑说明 (Data Management Guide)

本文档详细说明了本项目当前采用的**本地文件系统 (File System Access API)** 存储方案。

本项目**不依赖任何后端数据库**来存储用户档案和聊天记录，而是直接将数据**持久化存储在用户本地设备的文件夹中**。这种设计最大程度地保护了用户隐私，并模拟了“云数据库”的使用体验。

## 1. 核心机制：File System Access API (FSA)

我们利用浏览器原生的 [File System Access API](https://developer.mozilla.org/en-US/docs/Web/API/File_System_Access_API) 实现对本地文件系统的读写权限。

*   **按需授权**：用户在首次点击“开始使用”时，浏览器会弹出目录选择框，请求用户选择一个本地文件夹（如桌面的 `10k_data` 文件夹）。
*   **持久化句柄**：一旦授权，应用会持有该目录的操作句柄（Handle），并在会话期间持续读写。
*   **隐私安全**：所有数据仅存在于用户自己的电脑上，除非用户主动通过聊天发送图片给 AI 服务器处理，否则档案数据绝不离线。

## 2. 目录结构设计

当用户选定一个根目录（Root Directory）后，系统会自动创建以下层级结构：

```text
[用户选定的根目录]
├── user_123456/                 <-- 自动生成的随机用户ID文件夹
│   ├── profile.json             <-- 用户医美档案（核心数据）
│   └── chat_history/            <-- 聊天记录文件夹
│       ├── session_1710001.json <-- 单次对话记录
│       ├── session_1710002.json
│       └── ...
└── user_789012/                 <-- 如果模拟新用户登录，会生成新的ID文件夹
```

### 2.1 用户 ID (userId)
*   **生成规则**：每次应用初始化（或检测到无会话记录）时，系统会生成一个随机 ID（如 `user_839210`），模拟“新用户”场景。
*   **隔离性**：不同用户的数据完全隔离，互不干扰。

### 2.2 医美档案 (profile.json)
*   **存储位置**：`/{userId}/profile.json`
*   **更新策略**：**覆盖更新 (Merge & Overwrite)**
    *   当用户在“医美档案”页面点击保存时，系统先读取现有的 `profile.json`。
    *   将新提交的表单数据与旧数据合并。
    *   将合并后的完整 JSON 写回文件。
*   **数据结构定义 (Schema)**：
    ```json
    {
      "gender": "string",       // 性别 (例如: "female", "male")
      "age": "string",          // 年龄段 (例如: "25-30")
      "history": [              // 医美历史记录 (字符串数组)
        "string",               // 例如: "2023年做过光子嫩肤"
        "string"
      ],
      "photo": "string",        // 用户头像 (Base64 编码的图片数据, data:image/jpeg;base64,...)
      "updatedAt": "string"     // 最后更新时间 (ISO 8601 格式, 例如: "2023-10-27T10:00:00.000Z")
    }
    ```
    *注意：`photo` 字段可能包含较大的 Base64 字符串，建议后端在设计数据库时考虑存储容量或将其转存为对象存储 URL。*

### 2.3 聊天记录 (chat_history/)
*   **存储位置**：`/{userId}/chat_history/session_{timestamp}.json`
*   **写入时机**：用户每发送一条消息或 AI 回复一条消息，系统都会实时写入对应的 Session 文件。
*   **文件格式**：每个 JSON 文件代表一次完整的对话会话，包含对话标题、开始时间和消息列表。

## 3. 关键业务流程

### 3.1 启动与授权
1.  用户访问首页，点击“开始使用”。
2.  应用调用 `window.showDirectoryPicker()`。
3.  用户选择本地文件夹并点击“允许”读写。
4.  系统初始化 `DataService`，生成 `userId`，并自动创建文件夹结构。

### 3.2 档案保存
1.  用户在 `/profile` 页面填写信息。
2.  点击“保存”。
3.  调用 `dataService.saveUserProfile()`。
4.  直接写入本地 `profile.json` 文件（无下载弹窗，无网络请求）。

### 3.3 历史回溯
1.  用户点击右上角“历史记录”图标。
2.  系统遍历 `chat_history/` 目录下的所有 JSON 文件。
3.  解析文件元数据（标题、时间），按时间倒序展示在侧边栏。
4.  点击某条记录，读取该 JSON 内容并恢复到聊天界面。

## 4. 优势与限制

### 优势
*   **数据主权**：用户完全拥有自己的数据。
*   **零后端成本**：无需搭建数据库和存储服务。
*   **离线可用**：除了 AI 处理需要联网，查看历史记录和编辑档案均可离线进行。

### 限制
*   **浏览器兼容性**：仅支持 Chrome, Edge 等现代浏览器（不支持 Firefox, Safari）。
*   **权限重置**：页面彻底刷新后，浏览器出于安全考虑可能会断开文件句柄，下次操作时需用户再次确认授权（应用已做引导处理）。
